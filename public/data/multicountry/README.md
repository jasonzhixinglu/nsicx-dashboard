# multicountry-nsicx dashboard data export

This directory is the data feed for the multi-country NSICX dashboard.
Regenerated by `scripts/build_dashboard_export.py` from the production
DNS estimates (`output/dns_production/`), Consensus surveys, the SA CPI
panel, and the anchoring regressions. All values are pre-computed; the
dashboard does no estimation.

## Methodology

The DNS state estimates are produced under the **calendar-mode (no-strip)
measurement model**: each Consensus calendar-year forecast `t_k` enters
the observation equation directly with window
`[h1, h2] = [12(k-1)-(M-1), 12k-(M-1)]` (a 12-month window, possibly
extending into the past for `k=1` in non-January survey months `M`).
Under NSICX no-arbitrage the standard NS forward-average loading
`ns_loadings_forward(h1, h2, lambda)` handles negative `h1` cleanly,
and no realized-CPI data is required to construct the panel. This
replaces the previous YTD-stripping recipe (now archived at
`archive/stripping/`), which carried a `1/J` leverage factor at year-end
that contaminated short- and long-horizon residuals alike.

Full derivation (including the equivalence
`x(12;lambda)' F^{-(M-1)} beta_t = x_{[h1,h2]}(lambda)' beta_t`) is in
`methodology.pdf` distributed alongside this README.

## Layout

```
output/dashboard/
  README.md                        (this file)
  methodology.pdf                  design note (5 pages)
  manifest.json                    machine-readable index
  schema_version.txt               bare integer
  countries/
    {slug}/
      states.json                  filtered + smoothed L/S/C with SEs, plus lambda
      cpi.json                     CPI YoY series (full sample, SA)
      surveys.json                 Consensus survey rows for the snapshot vintages
      mle.json                     MLE summary (params, log-lik, T, etc.)
  cross_country/
    anchoring.json                 anchoring regressions (calmod-state-derived)
```

`{slug}` is the lowercase, underscored country name (`usa`, `south_korea`,
`uk`, etc.) — same convention used in `output/dns_production/`.

## Quick start

**Python:**

```python
import json
from pathlib import Path

ROOT = Path("output/dashboard")
manifest = json.load(open(ROOT / "manifest.json"))
print(manifest["pipeline_variant"], len(manifest["countries"]))

usa_states = json.load(open(ROOT / "countries" / "usa" / "states.json"))
print(usa_states["lambda"], len(usa_states["filtered"]))
```

**JavaScript / browser:**

```js
const manifest = await fetch("/dashboard/manifest.json").then(r => r.json());
const usa = await fetch("/dashboard/countries/usa/states.json").then(r => r.json());
console.log(usa.lambda, usa.filtered.length);
```

## File schemas

### `manifest.json`

```json
{
  "schema_version":   1,
  "pipeline_variant": "calendar_mode_no_strip",
  "generated_at":     "2026-05-...",
  "survey_periods":   ["2026-01", "2026-02", "2026-03", "2026-04"],
  "countries":        [{"slug": "usa", "name": "USA", "last_vintage": "2026-04",
                       "T": 304, "n_surveys": 6}, ...],
  "methodology_pdf":  "methodology.pdf"
}
```

### `countries/{slug}/states.json`

```json
{
  "country":  "USA",
  "lambda":   0.1542,
  "filtered": [{"d": "2001-01", "L": ..., "S": ..., "C": ...,
                "L_se": ..., "S_se": ..., "C_se": ...}, ...],
  "smoothed": [...]
}
```

- `d`: month string `"YYYY-MM"`.
- `L`, `S`, `C`: NS factors at the month, in **percent**.
- `L_se`, `S_se`, `C_se`: state filtering/smoothing standard errors. 95%
  CI = `± 1.96·SE`. Reflects filtering uncertainty only — parameter
  estimation uncertainty is not included.
- `lambda`: NS curvature parameter. Required to evaluate the curve at
  any horizon. Used by both filtered and smoothed factors.

### `countries/{slug}/cpi.json`

```json
{"country": "USA", "yoy_pct": [{"d": "2001-01", "v": 5.7138}, ...]}
```

`yoy_pct[*].v` is realized YoY CPI inflation in percent on the X-12-default
SA CPI level, i.e. `(log(SA_CPI_t) - log(SA_CPI_{t-12})) * 100`. Coverage
starts at 2001-01.

### `countries/{slug}/surveys.json`

```json
{"country": "USA",
 "survey_periods": ["2026-01", ..., "2026-04"],
 "rows": [{"survey_period": "2026-01", "survey_date": "2026-01-13",
           "source": "ST", "t1": 2.7, "t2": 2.5, ..., "t25": null}, ...]}
```

`source`: `"ST"` (monthly Consensus) or `"LT"` (semi-annual). `t1`..`t25`
are Consensus forecast horizons; some are `null` when the survey did not
include that horizon. The dashboard interprets per-source.

### `countries/{slug}/mle.json`

Verbatim copy of `output/dns_production/{slug}/mle_summary.json`. Includes
`params` (qL, qS, qC, lam, v1..v6), `bse` (asymptotic SEs from the MLE
Hessian), `loglik`, `nobs`.

### `cross_country/anchoring.json`

```json
{"main": {"USA": {"pi_qoq": {...}, ...}, ...},
 "raw_revisions": {"USA": {"pi_qoq": {...}, ...}, ...},
 "spec_legend": {...}}
```

Computed against the calmod-state filtered L/S/C. Two specs (`main` =
filter-implied long-end change; `raw_revisions` = raw long-end forecast
change in surveys) with `alpha`, `beta`, `p_beta`, `r2`, `T` per
country per RHS variable (`pi_qoq`, `pi_yoy`, `pi_sur`, and
`delta_ST`/`st_rev`).

## Conventions

- All dates are `"YYYY-MM"` strings unless explicitly a date.
- All percent values are in **percent** units (e.g. `2.5` means 2.5%).
- Slugs match `output/dns_production/{slug}/` — lowercase with
  underscores: `south_korea`, `uk`, `usa`, etc.
- Missing values are `null`.
- Numeric values rounded to 4 decimal places; underlying parquets carry
  full precision.
